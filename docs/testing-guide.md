# AMMOR Testing Guide

## Automated testing matrix

- Unit + regression: `bun test`
- Security-focused audit checks: `bun run test:audit`
- Pull request workflow: `.github/workflows/testing-workflow.yml`
- AMMOR product workflow gate job set (claims/cases/AI/evidence)
  - `.github/workflows/testing-workflow.yml` now auto-discovers AMMOR application workspaces and runs domain-specific test scripts when present:
    - required: `test:claims`, `test:cases`, `test:api`, `test:ai`, `test:dashboard`, `test:e2e`
    - optional: `test:evidence`, `test:human-review`, `test:ui`, `test`
    - strict enforcement default: `AMMOR_ENFORCE_PRODUCT_TEST_SCRIPTS=true`
    - set `AMMOR_ENFORCE_PRODUCT_TEST_SCRIPTS=false` only while AMMOR workspace onboarding is in progress.
- Visual regression + accessibility/UX gates:
  - Enable by setting repository variable `AMMOR_RUN_UX_GATES=true`
  - Strictly enforced scripts:
    - visual: `test:visual`, `test:visual-regression`, `test:ux`, `test:e2e:visual`
    - accessibility: `test:a11y`, `test:accessibility`, `test:wcag`, `test:axe`
  - Optional base URL: `AMMOR_UX_BASE_URL`
  - Strict enforcement default: `AMMOR_ENFORCE_UX_TEST_SCRIPTS=true`

## Roadmap and PM automation

- AMMOR roadmap automation is wired by `.github/workflows/ammor-roadmap-automation.yml`.
- Issue/PR labels can map to milestones via `.github/ammor-roadmap-automation.json`.
- Optional roadmap project sync supports GitHub Projects V2 when:
  - `AMMOR_ROADMAP_PROJECT_ID` is set in repository variables
  - `AMMOR_ROADMAP_TOKEN` is set as a repository secret

## Execution and evidence

- Security/compliance/testing artifact generation can be triggered manually from
  - **Actions → AMMOR Safety Gate Runner → Run workflow**
  - This dispatches:
    - `security-workflow.yml`
    - `testing-workflow.yml`
    - `deployment-safety-workflow.yml`
  - Compliance gate runs automatically after security workflow completion.
- Product flow and UX artifacts are uploaded to:
  - `qa-artifacts/ammor-product-tests/*`
  - `qa-artifacts/ammor-testing-aggregate.*`

## Mandatory PR test requirements

- `bun test` must pass.
- Required test files should include:
  - claim/case core logic tests,
  - API endpoint tests,
  - evidence upload path tests,
  - AI workflow transition tests.

## Optional high-value additions

- Login/auth flow tests for each portal role.
- Dashboard interaction tests for drag-and-drop and queue transitions.
- Accessibility and responsive checks in CI.
- Visual regression checks for screenshot deltas.

## Test reporting

- `qa-artifacts/testing-summary.json` and `.md` are generated by the testing workflow.
- Use these artifacts for PR readiness reviews and release sign-off.
