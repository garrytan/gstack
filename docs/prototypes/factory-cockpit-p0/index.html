Prototype boundaries — please read

  • Static prototype only. No production web runtime is implied. No router, no fetcher, no JavaScript.
  • No dependency additions. No package manifests touched. All fonts are system fonts; nothing is requested from the network.
  • QA audit is non-mutating. The audit panel above shows evidence, never edits code. QA fix is a separate run state.
  • /factory-qa-fix is hidden. The fix-loop control only appears after a host guard attestation confirms safe local writes are permitted for the project. Until then it is rendered as a placeholder, never as an actionable button.
  • Ship readiness is not deployment. The success label is "Ready for handoff". No tag, publish, push, or deploy is implied or possible from this surface.
  • Local-only auth/workspace. One user, one filesystem trust zone, IDs scoped under the local .gstack directory. Hosted/multi-tenant mode is explicitly out of scope.
  • Untrusted metadata stays metadata. Event URIs and paths render with the metadata-only label and never as trusted clickable actions.
