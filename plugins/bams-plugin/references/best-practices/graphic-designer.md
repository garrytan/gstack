# Graphic Designer Best Practices

## Responsibility: 아이콘 시스템 설계, 일러스트레이션 제작, AI 이미지 생성, 브랜드 그래픽, 에셋 관리

---

### 아이콘 시스템 설계 (design_icon_system)

**언제 참조:** 새 아이콘이 필요할 때, 또는 기존 아이콘 시스템을 확장할 때

**협업 대상:**
- design-director: 브랜드 가이드라인 및 아이콘 스타일 방향 확인
- design-system-agent: 아이콘 에셋을 디자인 시스템 레지스트리에 등록
- frontend-engineering: SVG 최적화 및 구현 방식 합의

**작업 절차:**
1. 기존 아이콘 세트(Heroicons, Lucide, Phosphor 등)와 비교하여 필요한 경우에만 커스텀 제작한다
2. 선 굵기, 모서리 처리, 크기 그리드(16/20/24/32px)를 표준화한다
3. outlined/filled/colored 변형을 정의한다
4. 아이콘 이름은 `noun-modifier` 패턴으로 통일한다 (예: `arrow-right`, `check-circle`)
5. SVG를 SVGO로 최적화하고 모든 아이콘에 title 요소 또는 aria-label을 추가한다

**산출물:** SVG 아이콘 파일 세트, 아이콘 스타일 가이드

**주의사항:**
- 최적화 후 파일 크기가 1KB를 초과하면 경로 단순화를 추가 적용한다
- 모든 아이콘에 접근성 속성(title, aria-label)을 추가하여 스크린 리더 접근성을 확보한다

### 일러스트레이션 제작 (create_illustration)

**언제 참조:** 빈 상태(Empty State), 온보딩, 에러 화면, 마케팅 소재 제작 시

**협업 대상:**
- design-director: 브랜드 아이덴티티 및 일러스트 스타일 방향 확인
- ux-designer: 각 화면 맥락(빈 상태, 에러 등)에서의 사용 시나리오 확인
- design-system-agent: 일러스트레이션 에셋을 레지스트리에 등록

**작업 절차:**
1. 제품 내 일러스트레이션이 필요한 맥락을 파악한다 (빈 상태, 온보딩, 에러, 성공 등)
2. 브랜드 아이덴티티(컬러 팔레트, 선 스타일, 캐릭터 유무)를 참조하여 스타일을 결정한다
3. 각 일러스트레이션이 텍스트 없이도 의미를 전달하는지 확인한다
4. SVG 형식으로 제작하고 SVGO로 최적화한다
5. 접근성을 위해 의미 있는 alt 텍스트를 포함한다

**산출물:** 일러스트레이션 SVG 파일, 맥락별 사용 가이드

**주의사항:**
- 일러스트레이션이 브랜드 톤과 일관되지 않으면 사용자 신뢰를 해친다 — 스타일 일관성을 최우선으로 한다
- 장식 목적의 일러스트레이션은 `aria-hidden="true"`로 처리한다

### AI 이미지 생성 (generate_ai_image)

**언제 참조:** 초안 생성 속도를 높이거나 반복 에셋 제작을 자동화할 때

**협업 대상:**
- design-director: AI 생성 결과가 브랜드 가이드라인에 부합하는지 검토 요청

**작업 절차:**
1. Recraft(벡터/일러스트 특화) 또는 Lunacy(UI/UX 에셋)를 사용한다 — 라이선스가 명확한 도구만 사용한다
2. 프롬프트를 구체적으로 작성한다 (스타일, 컬러, 구성 요소, 분위기를 포함)
3. 생성된 결과를 브랜드 가이드라인에 맞게 편집하고 정제한다
4. AI 생성 결과임을 파일 메타데이터나 레이어 이름에 표시한다
5. design-director의 검토를 받는다

**산출물:** 큐레이션된 AI 생성 에셋

**주의사항:**
- AI 생성 결과를 검토 없이 그대로 사용하지 않는다
- 라이선스가 불명확한 도구로 생성한 에셋은 절대 사용하지 않는다

---

### 에셋 관리 및 최적화 (manage_assets)

**언제 참조:** 제작한 에셋을 프로젝트에 반영하거나 기존 에셋을 최적화할 때

**협업 대상:**
- design-system-agent: 에셋을 디자인 시스템 레지스트리에 등록
- frontend-engineering: WebP 변환, srcset 구성, lazy loading 적용

**작업 절차:**
1. SVG는 viewBox와 aria-label을 포함하여 접근성을 확보한다
2. 래스터 이미지는 WebP 형식으로 변환하고 해상도별 srcset을 준비한다
3. 이미지 최적화 후 파일 크기를 확인한다 (아이콘 < 1KB, 일러스트 < 10KB, 사진 < 200KB)
4. 에셋을 카테고리별로 분류하고 design-system-agent의 레지스트리에 등록한다

**산출물:** 최적화된 에셋 파일, 레지스트리 등록 정보

**주의사항:**
- 에셋 이름 변경 시 frontend-engineering에게 통보한다 — 코드에서 파일 경로가 깨질 수 있다
- 에셋 최적화 전후 파일 크기를 기록하여 최적화 효과를 추적한다
