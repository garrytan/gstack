# UX Research Best Practices

## Responsibility: 사용자 조사 설계, 여정 맵핑, 사용성 리뷰

---

### 사용자 조사 설계 (design_user_research)

**언제 참조:** product-strategy로부터 사용자 조사 요청을 받았을 때, 또는 PRD의 사용자 가정을 검증할 때

**협업 대상:**
- product-strategy: 조사 목적 및 핵심 가설 확인, 조사 결과를 PRD에 통합
- product-analytics: 정량 데이터와 정성 분석 교차 검증

**작업 절차:**
1. 조사 목적(탐색/검증/평가)을 명확히 하고 방법론을 선택한다
2. 정성(인터뷰, 관찰)과 정량(설문, 로그 분석) 조사를 목적에 맞게 조합한다
3. 유도 질문을 배제하고 개방형 질문을 우선 배치한 질문지를 작성한다
4. 조사 대상 선정 기준과 표본 크기의 근거를 명시한다

**산출물:** 리서치 계획서 (조사 목적, 방법론, 대상 선정 기준, 질문지/태스크 시나리오, 일정)

**주의사항:**
- 개인 취향이 아닌 사용자 데이터와 원칙에 근거하여 판단한다
- "사용자는 ~할 것이다"라는 가정은 검증 전에 사용하지 않는다
- 본 에이전트(ux-research)는 기획 단계 리서치 담당 — 구현 후 세부 UX 설계는 ux-designer 담당

### 사용자 여정 맵핑 (map_user_journey)

**언제 참조:** PRD의 사용자 시나리오를 구체화할 때, 또는 기존 플로우의 이탈 원인을 분석할 때

**협업 대상:**
- ux-designer: 여정 맵을 와이어프레임 설계의 입력으로 전달
- frontend-engineering: 화면 구조와 라우팅 확인으로 실제 흐름과 대조
- product-analytics: 이탈 지점 데이터와 여정 맵 교차 분석

**작업 절차:**
1. 사용자 목표를 먼저 정의한 후 단계별 행동을 순서대로 나열한다
2. 각 단계에서 사용자의 생각, 감정, 행동, 터치포인트를 구분하여 기록한다
3. 마찰 지점(Friction Point)과 이탈 지점(Drop-off Point)을 명확히 표시한다
4. 기존 코드베이스의 화면 구조와 라우팅을 Glob, Read로 확인하여 실제 흐름과 대조한다
5. 개선 기회를 우선순위 순으로 정렬한다

**산출물:** 사용자 여정 맵 (페르소나, 목표, 단계별 행동/감정/터치포인트, 페인 포인트, 개선 기회)

**주의사항:**
- 여정 맵은 ux-designer의 와이어프레임 설계 입력으로 직접 사용된다 — 충분히 구체적으로 작성한다
- 감정 곡선을 포함하여 어느 지점에서 사용자 만족도가 급락하는지 시각화한다

### 사용성 휴리스틱 평가 (evaluate_usability)

**언제 참조:** 구현된 화면 또는 프로토타입의 사용성 문제를 검토할 때

**협업 대상:**
- ux-designer: 사용성 문제를 UX 설계 개선에 반영
- frontend-engineering: UI 구현 상태와 사용성 리뷰 연계
- qa-strategy: 사용성 문제를 QA 체크리스트에 추가

**작업 절차:**
1. Nielsen 10대 휴리스틱 원칙을 체크리스트로 활용하여 누락 없이 평가한다
2. 각 문제에 심각도를 부여한다: 치명적/중대/경미/개선희망
3. 문제 지적과 함께 구체적인 개선 방안을 반드시 제시한다
4. 실제 구현된 컴포넌트와 UI 코드를 Read하여 설계 의도와 구현 간 괴리를 탐지한다

**산출물:** 사용성 리뷰 보고서 (평가 대상, 발견 사항 목록, 심각도별 요약, 즉시 조치 필요 항목)

**주의사항:**
- 심각도 "치명적" 항목은 릴리즈 전 반드시 해결해야 한다 — release-quality-gate에 통보한다
- 개선 방안 없이 문제만 나열하는 것은 허용되지 않는다

### 페르소나 구축 (build_persona)

**언제 참조:** 새 제품 또는 새 피처의 타깃 사용자를 정의할 때

**협업 대상:**
- product-strategy: 페르소나를 PRD의 타깃 사용자 정의에 반영
- ux-designer: 페르소나를 와이어프레임 설계의 기준점으로 활용
- business-analysis: 페르소나 기반 요구사항 우선순위 검토

**작업 절차:**
1. 리서치 데이터(인터뷰, 설문, 로그)에서 행동 패턴 클러스터를 도출한다
2. 각 클러스터를 대표하는 페르소나를 정의한다 (이름, 배경, 목표, 불만, 행동 패턴)
3. 페르소나 간 우선순위를 정하고 Primary/Secondary를 구분한다
4. 각 페르소나의 제품 사용 시나리오를 작성한다

**산출물:** 페르소나 문서 (페르소나 프로필, 시나리오, Primary/Secondary 구분)

**주의사항:**
- 페르소나는 실제 데이터에 기반해야 한다 — 데이터 없이 만든 페르소나는 "proto-persona"로 명시한다
- 페르소나는 최대 3개까지만 작성한다 — 그 이상은 집중도가 떨어진다
