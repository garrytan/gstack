# Resource Optimizer Best Practices

## Responsibility: 모델 선택, 병렬화 전략, 비용/속도/품질 밸런싱

---

### 모델 선택 (select_model)

**언제 참조:** pipeline-orchestrator가 파이프라인 시작 시 각 에이전트의 모델을 결정할 때

**협업 대상:**
- pipeline-orchestrator: 모델 전략을 파이프라인 실행 계획에 반영

**작업 절차:**
1. 파이프라인 유형(feature/hotfix/dev)과 각 Phase의 복잡도를 분석한다
2. 태스크 특성별 모델을 매핑한다:
   - opus: 전략적 판단, 복잡한 설계, 트레이드오프 분석 (product-strategy, design-director)
   - sonnet: 구현, 분석, 조율, 보고 (대부분의 에이전트)
   - haiku: 단순 반복 작업, 형식 변환, 빠른 체크
3. 품질 요구 수준(Critical 기능 vs. 내부 도구)에 따라 모델 등급을 조정한다
4. 예상 비용을 산출하고 예산 초과 시 다운그레이드 후보를 제안한다

**산출물:** 에이전트별 모델 할당표 (에이전트명, 추천 모델, 선택 근거)

**주의사항:**
- 모델 다운그레이드 시 품질 저하 위험을 반드시 명시한다
- hotfix 파이프라인은 속도를 우선하여 sonnet 또는 haiku를 기본으로 권장한다
- opus는 비용이 높으므로 전략적 판단이 필요한 에이전트에만 제한적으로 사용한다

### 병렬화 설계 (design_parallelization)

**언제 참조:** pipeline-orchestrator가 대규모 파이프라인(Phase 3개 이상)의 실행 전략을 수립할 때

**협업 대상:**
- pipeline-orchestrator: 병렬화 전략을 실행 계획에 반영
- cross-department-coordinator: 병렬 실행 시 부서 간 인터페이스 사전 합의 필요 여부 확인

**작업 절차:**
1. Phase 간 의존성 그래프를 분석하여 병렬 실행 가능한 구간을 식별한다
2. 독립적인 에이전트 그룹을 묶어 병렬 실행 배치를 구성한다
3. 병렬 실행 시 공유 리소스(파일 쓰기 경합)를 점검하고 충돌 지점을 제거한다
4. 병렬화 효율(시간 단축 비율)과 복잡도 증가를 비교하여 최적 구간을 결정한다

**산출물:** 병렬화 실행 계획 (병렬 그룹, 순차 실행 구간, 의존성 다이어그램)

**주의사항:**
- 병렬 실행 중 하나가 실패하면 의존하는 그룹을 즉시 중단하는 장치를 포함한다
- 파일 쓰기 충돌을 방지하기 위해 각 에이전트의 출력 경로를 사전에 분리한다

### 비용 분석 (analyze_cost)

**언제 참조:** 파이프라인 비용 검토 요청 시, 또는 예산 초과 경고 시

**협업 대상:**
- pipeline-orchestrator: 비용 분석 결과를 파이프라인 계획에 반영
- executive-reporter: 비용 추적 데이터를 보고서에 포함

**작업 절차:**
1. 파이프라인의 예상 에이전트 호출 횟수와 각 모델의 토큰 사용량을 추정한다
2. 모델별 단가를 적용하여 총 예상 비용을 산출한다
3. 이전 유사 파이프라인의 실제 비용과 비교한다
4. 비용 절감 방안(모델 다운그레이드 후보, 불필요한 Step 스킵)을 제안한다

**산출물:** 비용 분석 보고서 (예상 비용, 모델별 분해, 절감 방안)

**주의사항:**
- 비용 추정은 반드시 신뢰구간 범위를 포함한다 (예: "$2~4 추정")
- 품질 저하 없이 비용을 절감할 수 있는 경우에만 다운그레이드를 권장한다
