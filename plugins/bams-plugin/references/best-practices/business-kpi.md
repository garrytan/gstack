# Business KPI Best Practices

## Responsibility: 매출/비용 영향 분석, 경영진 보고 수치 요약, ROI 분석, 목표 추적 및 예측, 비즈니스 이상 탐지

---

### 매출/비용 영향 분석 (analyze_revenue_cost_impact)

**언제 참조:** 피처 릴리즈 후 또는 product-analytics로부터 영향 평가 요청을 받았을 때

**협업 대상:**
- product-analytics: 제품 지표에서 사업 지표로의 연결 고리 확인
- executive-reporter: 분석 결과를 경영진 보고서에 포함
- product-strategy: 사업 성과를 다음 로드맵 우선순위에 반영

**작업 절차:**
1. 직접 효과와 간접 효과를 분리하여 분석한다 (예: 전환율 개선 → 매출 직접 증가, 이탈률 감소 → 장기 LTV 간접 증가)
2. 고정비와 변동비를 구분하여 비용 분석을 수행한다
3. 외부 요인(계절성, 시장 변화)의 영향을 보정한다
4. 영향 수치에 신뢰구간을 포함한다 (점 추정치만 보고하지 않는다)

**산출물:** 매출/비용 영향 분석 보고서 (직접/간접 효과 분리, 외부 요인 보정, 신뢰구간)

**주의사항:**
- 모든 사업 수치는 출처와 산출 근거를 함께 제시한다
- "매출이 늘었습니다" 대신 "피처 X 도입 후 D+7 전환율이 2.3%p 개선되어 월 매출 약 N만원 증가 추정 (95% CI: N±M만원)"처럼 구체화한다

### 경영진 보고 수치 요약 (summarize_executive_metrics)

**언제 참조:** executive-reporter의 경영진 보고 요청 시, 또는 이사회/투자자 보고 준비 시

**협업 대상:**
- executive-reporter: 보고서 통합 및 배포
- product-analytics: 제품 지표 데이터 수신

**작업 절차:**
1. 핵심 KPI를 의사결정에 최적화된 형태로 1페이지 이내로 요약한다
2. 신호등 체계(Green/Yellow/Red)로 각 KPI의 목표 달성 상태를 표시한다
3. 이전 기간 대비 변화율과 목표 대비 달성률을 함께 제시한다
4. 의사결정이 필요한 항목을 별도 섹션으로 분리한다

**산출물:** 경영진 보고 요약 (1페이지 핵심 KPI 대시보드, 의사결정 필요 항목)

**주의사항:**
- 경영진 보고는 1페이지 이내 원칙을 준수한다 — 세부 데이터는 별첨으로 제공한다
- 기술 용어를 배제하고 비즈니스 언어로 작성한다

### ROI 분석 (analyze_roi)

**언제 참조:** 기능 개발 투자 대비 사업 성과를 산출할 때

**협업 대상:**
- project-governance: 개발 비용(공수, 인프라 비용) 데이터 수신
- product-analytics: 기능 사용 데이터 및 전환 기여 분석 협업

**작업 절차:**
1. 투자 비용을 산출한다: 개발 공수(인건비), 인프라 비용, 유지보수 비용
2. 수익 기여를 산출한다: 전환율 개선, 이탈률 감소, 신규 기능 수익화
3. ROI를 계산한다: (수익 기여 - 투자 비용) / 투자 비용 × 100%
4. 투자 회수 기간(Payback Period)을 산출한다

**산출물:** ROI 분석 보고서 (투자 비용 분해, 수익 기여 분석, ROI, 투자 회수 기간)

**주의사항:**
- ROI 분석은 최소 3개월 데이터가 축적된 후 수행한다 — 초기 데이터는 노벨티 효과로 왜곡될 수 있다
- 비용 산출 시 직접 비용(개발)뿐 아니라 기회 비용(다른 기능을 못 만든 것)도 고려한다

### 목표 추적 및 예측 (track_and_forecast_targets)

**언제 참조:** OKR/KPI 달성률을 주기적으로 추적하거나 분기 목표 달성 가능성을 예측할 때

**협업 대상:**
- product-analytics: 실제 지표 데이터 수신
- executive-reporter: 추적 결과를 정기 보고에 포함

**작업 절차:**
1. 현재 달성률을 목표 대비 계산한다
2. 현재 트렌드가 지속될 경우 목표 달성 가능성을 예측한다
3. 목표 달성이 어려울 경우 갭을 줄이기 위한 방안을 제안한다
4. 예측치는 신뢰구간과 전제 조건을 명시한다

**산출물:** 목표 추적 보고서 (현재 달성률, 달성 예측, 갭 분석, 개선 방안)

**주의사항:**
- 목표 달성이 어려울 때 수치를 미화하거나 정의를 임의로 변경하지 않는다
- 예측치는 반드시 신뢰구간을 포함한다 — 단일 점 추정치로 의사결정하면 위험하다
