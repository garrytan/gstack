# Executive Reporter Best Practices

## Responsibility: 파이프라인 상태 집계, 대시보드 생성, 경영진 보고서 작성

---

### 실시간 상태 집계 (aggregate_realtime_status)

**언제 참조:** pipeline-orchestrator로부터 Phase 완료 상태 기록 요청을 받았을 때, 또는 파이프라인 진행 중 현황 조회 요청 시

**협업 대상:**
- pipeline-orchestrator: 집계 결과를 Phase 게이트 판단의 근거로 제공
- product-analytics: 정량 지표 교차 검증 협업
- business-kpi: 사업 성과 지표 통합

**작업 절차:**
1. `.crew/artifacts/pipeline/` 하위 이벤트 로그에서 pipeline_start/pipeline_end, agent_start/agent_end 이벤트를 파싱한다
2. 진행률(완료 Step / 전체 Step), 소요 시간(Phase별), 블로커 현황을 계산한다
3. 신호등 체계(Green/Yellow/Red)로 전체 상태를 분류한다
4. tracking 파일에 Phase 완료 상태를 기록한다

**산출물:** Phase 상태 요약 (진행률, 소요 시간, 블로커, 신호등 상태)

**주의사항:**
- 주관적 표현("잘 진행 중") 대신 수치 기반 지표를 사용한다
- 이벤트 로그가 누락된 경우 추정치임을 명시하고 에러로 처리하지 않는다

### 대시보드 생성 (generate_dashboard)

**언제 참조:** /bams:status 커맨드 실행 시, 또는 경영진 현황 보고 요청 시

**협업 대상:**
- project-governance: 일정 계획 대비 실적 데이터 수신
- business-kpi: KPI 달성률 데이터 수신

**작업 절차:**
1. 모든 활성 파이프라인의 이벤트 로그를 스캔한다
2. 파이프라인별 상태를 신호등 체계로 분류하고 수치를 집계한다
3. 완료/진행중/블로커 현황을 한눈에 파악 가능한 형태로 요약한다
4. 의사결정이 필요한 항목을 별도 섹션으로 분리하여 가시성을 확보한다

**산출물:** 실시간 대시보드 문서 (파이프라인 상태 테이블, 블로커 목록, 의사결정 필요 항목)

**주의사항:**
- 대시보드는 의사결정자가 3분 내에 전체 상황을 파악할 수 있는 수준으로 작성한다
- 세부 내용은 링크로 연결하고 대시보드 자체는 요약만 포함한다

---

### 회고용 정량 데이터 수집 (collect_retro_metrics)

**언제 참조:** pipeline-orchestrator가 파이프라인 완료 후 회고 데이터를 요청할 때

**협업 대상:**
- pipeline-orchestrator: 수집된 데이터를 회고 분석의 입력으로 제공
- product-analytics: 품질 지표 및 사용자 영향 데이터 교차 확인
- business-kpi: 사업 성과 지표 통합

**작업 절차:**
1. 이벤트 로그에서 총 소요 시간과 Phase별 소요 시간을 계산한다
2. Step 성공률, 재시도 횟수, 에이전트별 호출 통계를 집계한다
3. 이전 3회 파이프라인 데이터와 비교하여 트렌드를 분석한다
4. 품질 지표(결함 수, 테스트 통과율, 게이트 통과율)를 포함한다
5. 수집된 데이터를 pipeline-orchestrator에게 구조화된 형태로 전달한다

**산출물:** 회고용 정량 데이터 패키지 (소요 시간, 성공률, 재시도 횟수, 이전 대비 트렌드)

**주의사항:**
- 이전 3회 데이터가 없으면 "기준 데이터 없음"으로 표시하고 현재 회차부터 기록을 시작한다
- 이상값(outlier)은 사유와 함께 별도 표시한다

---

### 주간/스프린트 요약 (generate_periodic_summary)

**언제 참조:** /bams:weekly 또는 /bams:sprint 커맨드 실행 시

**협업 대상:**
- project-governance: 스프린트 목표 대비 실적 데이터 수신
- business-kpi: 사업 KPI 달성률 수신
- product-analytics: 제품 지표 요약 수신

**작업 절차:**
1. 해당 기간의 완료 태스크, 진행 중 태스크, 블로커를 집계한다
2. 스프린트 목표 대비 실적을 계산한다
3. 품질 지표(테스트 통과율, 결함 발생률)를 포함한다
4. 다음 스프린트의 예상 리스크와 의사결정 필요 항목을 제시한다

**산출물:** 주간/스프린트 요약 보고서 (성과, 블로커, 품질 지표, 다음 기간 전망)

**주의사항:**
- 요약 보고서는 경영진이 읽는 문서이므로 기술 용어를 최소화하고 비즈니스 언어로 작성한다
- 수치에 맥락을 함께 제공한다 (예: "테스트 통과율 87% — 이전 주 대비 +5%p")
